## Design Context

This project has captured design context via `/impeccable`. Read these before any UI work:

- **`PRODUCT.md`** — strategic context (source of truth for design decisions).
- **`DESIGN.md`** — visual system (colors, typography, components, layout).

Quick reference:
- **Register:** brand (marketing site — design is the product) · **Platform:** web
- **Primary audience:** dental practitioners / clinical partners (B2B). Patients are secondary proof.
- **Primary CTA:** Submit a trial case · **Secondary CTA:** Book a design meeting
- **Personality:** precision-engineered, clinical — confidence through exactness, warm through competence not gimmick
- **Emotional goal:** clinical confidence and trust
- **Avoid:** consumer DTC/gimmicky, generic dental templates, sterile/lifeless, generic SaaS slop
- **Accessibility target:** WCAG 2.2 AA
